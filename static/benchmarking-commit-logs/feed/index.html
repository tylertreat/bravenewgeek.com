“For a reason I have yet to figure out, the latency for 1MB messages is roughly an order of magnitude faster when fsync is enabled after the 95th percentile, which seems counterintuitive”

My suspicion is that fsync will prevent the kernel buffer to grow too large, so you get more constant latency numbers. Without fsync, the kernel buffer could grow large and when the kernel does flush it on disk, that can cause these spikes. Running strace tool may be able to prove that.
