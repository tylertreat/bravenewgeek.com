]]> By: chris https://bravenewgeek.com/abstraction-considered-harmful/#comment-14097 Sun, 28 Feb 2016 02:12:58 +0000 https://bravenewgeek.com/abstraction-considered-harmful/#comment-14097 That’s a good point of view. I tend to delay abstraction a bit. I’m a fan of continuous refactoring so even though I try to write the best code I can on the first shot, I assume it can and will be improved later. This is especially true with abstraction. I’ll accept duplicating code 2 or 3 times before I feel like I understand the problem well enough to abstract it in some cases. Even then, reality will require it to change and that should be easy. On the other hand, premature or bad abstraction seems to be a recipe for needless, self-propagating complexity.
